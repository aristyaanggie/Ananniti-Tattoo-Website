# Dokumentasi Project Ananniti Tattoo Bali

Project ini adalah website production untuk **Ananniti Tattoo Bali**, sebuah studio tato profesional di Bali yang menawarkan layanan custom tattoo design dan perawatan.

## Struktur Dokumentasi

Dokumentasi project tersimpan dalam folder `.ai/` dengan struktur berikut:

### Root Level
- **ai-rules.md**: Aturan dan panduan untuk AI development
- **README.md**: Dokumentasi project (file ini)
- **project.md**: Deskripsi project dan objectives
- **design.md**: Design system dan visual guidelines
- **architecture.md**: Arsitektur sistem dan struktur application
- **roadmap.md**: Roadmap product dan timeline
- **coding-rules.md**: Coding standards dan best practices
- **folder-structure.md**: Struktur folder project
- **review-checklist.md**: Checklist untuk code review
- **tech-stack.md**: Technology stack yang digunakan
- **ui-components.md**: Daftar UI components
- **flowchart.md**: Flowchart aplikasi
- **erd.md**: Entity Relationship Diagram
- **database-schema.md**: Database schema dan migrations
- **api-plan.md**: API endpoints documentation
- **deployment.md**: Deployment strategy dan process

### Context Folder (`context/`)
Berisi detailed guidelines untuk design dan implementation:
- **design-system.md**: Design system details
- **spacing.md**: Spacing dan layout guidelines
- **typography.md**: Typography standards
- **animation.md**: Animation guidelines
- **assets.md**: Assets management
- **responsive.md**: Responsive design guidelines
- **copywriting.md**: Copywriting style guide
- **reusable-components.md**: Reusable components documentation
- **naming-convention.md**: Naming conventions
- **ui-philosophy.md**: UI philosophy dan principles

### Prompts Folder (`prompts/`)
Berisi prompt templates untuk berbagai task:
- **first-prompt.md**: Initial project setup prompt
- **page-prompt.md**: Page creation prompt template
- **component-prompt.md**: Component creation prompt template
- **review-prompt.md**: Code review prompt template
- **bugfix-prompt.md**: Bug fix prompt template
- **refactor-prompt.md**: Refactoring prompt template
- **commit-prompt.md**: Commit message prompt template
- **deployment-prompt.md**: Deployment prompt template

### Todos Folder (`todos/`)
Project tracking dan progress:
- **00-project-setup.md**: Project setup checklist
- **progress.md**: Overall project progress

### Journal Folder (`journal/`)
Project history dan decisions:
- **sprint-00.md**: Sprint 00 journal
- **decision-log.md**: Major decisions log
- **known-issues.md**: Known issues dan workarounds
- **change-log.md**: Change log

## Cara Menggunakan Dokumentasi

1. **Sebelum coding**: Baca dokumentasi yang relevan (project.md, architecture.md, coding-rules.md)
2. **Saat development**: Ikuti guidelines dari design.md, ui-philosophy.md, dan coding-rules.md
3. **Saat review**: Gunakan review-checklist.md sebagai reference
4. **Saat deployment**: Ikuti deployment.md
