# Aturan AI

Dokumen ini mendefinisikan aturan dan panduan untuk AI dalam mengembangkan project Ananniti Tattoo Bali.

## Prinsip Utama

1. **Ikuti Dokumentasi**: Seluruh implementasi WAJIB mengikuti dokumentasi project yang telah ditetapkan.
2. **Bahasa**: Analisis, laporan, review, dan komunikasi menggunakan Bahasa Indonesia. Penamaan kode tetap Bahasa Inggris.
3. **Code Style**: Ikuti Laravel, Blade, dan Tailwind CSS best practice.
4. **No Premature Optimization**: Jangan membuat improvisasi di luar scope sprint.
5. **Clean Architecture**: Pertahankan struktur project tetap bersih dan scalable.

## Panduan Sprint

- Kerjakan hanya task pada sprint yang ditentukan.
- Jangan mengerjakan task sprint berikutnya.
- Jika menemukan ide atau improvement, tuliskan pada bagian Rekomendasi, jangan langsung implementasi.

## Kualitas Kode

- Tidak ada duplicate folder atau file.
- Penamaan folder dan file konsisten.
- Tidak ada file kosong.
- Build project berhasil tanpa error.
- Tidak ada warning di terminal.

## Dokumentasi

- Seluruh dokumentasi harus memiliki placeholder yang jelas.
- Jangan membuat file kosong.
- Setiap file markdown minimal memiliki: Judul, Deskripsi singkat, Placeholder.

## Version Control

- Commit hanya ketika diminta user.
- Gunakan commit message yang deskriptif dan singkat.
- Jangan force push atau melakukan operasi destruktif tanpa approval.

## Batasan

Tidak boleh membuat:
- Landing Page
- Navbar, Hero, About, Services, Gallery, Shop, Footer
- Blade Component UI
- Layout Utama
- Styling UI
- Business Logic
- Reusable Component

Sprint ini hanya membangun fondasi project.
